219 -- Estructura Básica para Sobrecargar Operadores --
220 En C++, los operadores se sobrecargan utilizando funciones miembro o funciones amigas.
221 La estructura básica para sobrecargar un operador es la siguiente:
222
223 cpp
224 tipo_retorno operatorop(const tipo& operando) {
225 // Implementación de la operación
226 // Puedes acceder a los miembros de la clase y a los miembros de operando
227 // Devuelve el resultado deseado
228 }
229 Donde:
230 operatorop es el operador que deseas sobrecargar (por ejemplo, +, -, *, /, <<, >>, etc.).
231 tipo_retorno es el tipo de dato que devuelve la operación.
232 tipo es el tipo de dato del operando con el que estás trabajando.
233
234 --- Sobrecarga del operador << ---
235 En C++, cuando sobrecargas el operador << para una clase, lo haces generalmente como una
236 función amiga (friend function) de esa clase. Esto tiene varias razones y beneficios:
237
238 -> Acceso a miembros privados: Como el operador << necesita acceder a los miembros privados
239 de la clase Fraccion para imprimirlos, se declara como amiga para tener acceso directo a
240 estos miembros, incluso si son privados.
241
242 -> Flujo de salida (ostream): El operador << está definido para trabajar con objetos que son
243 capaces de realizar operaciones de salida, representados por std::ostream. Esta clase
244 proporciona operaciones de salida comunes como operator<< para escribir datos en diversos
245 destinos (como la consola, archivos, etc.).
246
247 -> Referencia constante a Fraccion: El segundo parámetro de la función sobrecargada operator<<
248 es una referencia constante const Fraccion& frac. Esta referencia constante asegura que el
249 objeto frac pasado como argumento no se modificará dentro de la función operator<<.
