449 -- Estructura Básica para Sobrecargar Operadores --
450 En C++, los operadores se sobrecargan utilizando funciones miembro o funciones amigas.
451 La estructura básica para sobrecargar un operador es la siguiente:
452
453 cpp
454 tipo_retorno operatorop(const tipo& operando) {
455 // Implementación de la operación
456 // Puedes acceder a los miembros de la clase y a los miembros de operando
457 // Devuelve el resultado deseado
458 }
459 Donde:
460 operatorop es el operador que deseas sobrecargar (por ejemplo, +, -, *, /, <<, >>, etc.).
461 tipo_retorno es el tipo de dato que devuelve la operación.
462 tipo es el tipo de dato del operando con el que estás trabajando.
463
464 --- Sobrecarga del operador << ---
465 En C++, cuando sobrecargas el operador << para una clase, lo haces generalmente como una
466 función amiga (friend function) de esa clase. Esto tiene varias razones y beneficios:
467
468 -> Acceso a miembros privados: Como el operador << necesita acceder a los miembros privados
469 de la clase Fraccion para imprimirlos, se declara como amiga para tener acceso directo a
470 estos miembros, incluso si son privados.
471
472 -> Flujo de salida (ostream): El operador << está definido para trabajar con objetos que son
473 capaces de realizar operaciones de salida, representados por std::ostream. Esta clase
474 proporciona operaciones de salida comunes como operator<< para escribir datos en diversos
475 destinos (como la consola, archivos, etc.).
476
477 -> Referencia constante a Fraccion: El segundo parámetro de la función sobrecargada operator<<
478 es una referencia constante const Fraccion& frac. Esta referencia constante asegura que el
479 objeto frac pasado como argumento no se modificará dentro de la función operator<<.
480
