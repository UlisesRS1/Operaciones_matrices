340 -- Estructura Básica para Sobrecargar Operadores --
341 En C++, los operadores se sobrecargan utilizando funciones miembro o funciones amigas.
342 La estructura básica para sobrecargar un operador es la siguiente:
343
344 cpp
345 tipo_retorno operatorop(const tipo& operando) {
346 // Implementación de la operación
347 // Puedes acceder a los miembros de la clase y a los miembros de operando
348 // Devuelve el resultado deseado
349 }
350 Donde:
351 operatorop es el operador que deseas sobrecargar (por ejemplo, +, -, *, /, <<, >>, etc.).
352 tipo_retorno es el tipo de dato que devuelve la operación.
353 tipo es el tipo de dato del operando con el que estás trabajando.
354
355 --- Sobrecarga del operador << ---
356 En C++, cuando sobrecargas el operador << para una clase, lo haces generalmente como una
357 función amiga (friend function) de esa clase. Esto tiene varias razones y beneficios:
358
359 -> Acceso a miembros privados: Como el operador << necesita acceder a los miembros privados
360 de la clase Fraccion para imprimirlos, se declara como amiga para tener acceso directo a
361 estos miembros, incluso si son privados.
362
363 -> Flujo de salida (ostream): El operador << está definido para trabajar con objetos que son
364 capaces de realizar operaciones de salida, representados por std::ostream. Esta clase
365 proporciona operaciones de salida comunes como operator<< para escribir datos en diversos
366 destinos (como la consola, archivos, etc.).
367
368 -> Referencia constante a Fraccion: El segundo parámetro de la función sobrecargada operator<<
369 es una referencia constante const Fraccion& frac. Esta referencia constante asegura que el
370 objeto frac pasado como argumento no se modificará dentro de la función operator<<.
371
372 -> std::ostream& operator<<(std::ostream& os, const Fraccion& frac) // Estructura
